Vestiaire Collective Scraper

Advanced scraping with Scrapfly.io integration and limitation avoidance

GET /vestiaire

Search Vestiaire Collective products with enhanced reliability

GET /vestiaire?search=chanel%20bag&page=1&items_per_page=10&min_price=100&max_price=1000&country=uk
Parameters:
  • search - Search query (default: handbag)
  • page - Page number (default: 1)
  • items_per_page - Items per page (default: 50)
  • min_price - Minimum price filter
  • max_price - Maximum price filter
  • country - Country domain (default: uk)

Advanced Limitation Avoidance Features

Adaptive Rate Limiting

Dynamic rate adjustment based on success rates

  • Base limit: 20 requests/minute
  • Automatic backoff on failures
  • Per-client rate tracking
  • Success rate adaptation
Intelligent Caching

Smart caching to reduce API calls

  • 15-minute cache duration
  • Hit/miss tracking
  • Performance statistics
  • Automatic invalidation
Circuit Breaker Pattern

Prevent cascade failures

  • 3 failures threshold
  • 2-minute recovery timeout
  • Half-open state testing
  • Automatic service isolation
Retry Logic with Backoff

Handle transient failures gracefully

  • 3 maximum retry attempts
  • Exponential backoff (1s, 2s, 4s)
  • Configurable delays
  • Graceful error handling
How It Works

The limitation avoidance system automatically protects your API from rate limiting, service failures, and performance issues. When you make a request, it first checks the cache, then applies rate limiting, uses circuit breaker protection, and retries with exponential backoff if needed.
